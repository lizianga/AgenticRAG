# AgenticRAG 项目分析

## 技术栈

| 技术/框架 | 用途 | 来源 |
|---------|------|------|
| Python | 主要开发语言 | 项目代码 |
| LangChain | LLM应用构建框架 | model/factory.py, rag/rag_service.py |
| Chroma | 向量数据库 | rag/vector_store.py |
| OpenAI API | 模型调用接口 | model/factory.py |
| SiliconFlow | 模型API提供者 | model/factory.py |
| Qwen/Qwen3-8B | 聊天模型 | config/rag.yml |
| BAAI/bge-m3 | 嵌入模型 | config/rag.yml |

## 主要模块及功能

### 1. RAG 服务模块 (`rag/rag_service.py`)

**功能**：
- 实现检索增强生成(RAG)功能，从向量存储中检索相关文档并生成回答
- 提供文档检索接口，根据用户查询获取相关文档
- 构建提示词模板，结合检索到的文档生成回答

**核心组件**：
- `RagSummariceService` 类：管理RAG流程，包括文档检索和回答生成
- 向量存储：使用Chroma作为底层向量数据库
- 提示词模板：加载并使用预定义的提示词

### 2. 向量存储模块 (`rag/vector_store.py`)

**功能**：
- 管理文档的向量存储和检索
- 支持PDF和TXT文档的加载和处理
- 文档分块和向量化
- 增量更新机制，避免重复处理文档

**核心组件**：
- `VectorStoreService` 类：管理向量存储和文档处理
- 文档加载器：支持PDF和TXT格式
- 文本分割器：将文档分割成适当大小的块
- MD5校验：避免重复处理文档

### 3. 代理模块 (`agent/react_agent.py`)

**功能**：
- 构建基于React模式的代理
- 集成多种工具，提供综合问答能力
- 支持流式输出回答

**核心组件**：
- `ReactAgent` 类：管理代理的创建和执行
- 工具集成：包括RAG摘要、天气查询、用户位置获取等
- 中间件：监控工具使用、日志记录、提示词切换

### 4. 模型工厂模块 (`model/factory.py`)

**功能**：
- 管理LLM和嵌入模型的创建
- 统一模型配置和初始化

**核心组件**：
- `BaseModelFactory` 抽象类：定义模型工厂接口
- `ChatModelFactory` 类：创建聊天模型
- `EmbeddingsFactory` 类：创建嵌入模型

### 5. 工具模块 (`agent/tools/agent_tools.py`)

**功能**：
- 提供各种工具，扩展代理的能力
- 支持RAG摘要、天气查询、用户信息获取等功能

**核心工具**：
- `rag_summarize`：从向量存储中检索参考资料
- `get_weather`：获取指定城市的天气
- `get_user_location`：获取用户所在城市
- `get_user_id`：获取用户ID
- `get_current_month`：获取当前月份
- `fetch_external_data`：从外部系统获取用户使用记录
- `fill_context_for_report`：为报告生成注入上下文信息

### 6. 配置模块 (`config/`)

**功能**：
- 管理项目配置，包括模型选择、API密钥等
- 提供统一的配置访问接口

**配置文件**：
- `agent.yml`：代理配置，包括模型选择和API密钥
- `rag.yml`：RAG配置，包括模型选择
- `chroma.yml`：Chroma向量数据库配置
- `prompts.yml`：提示词配置

### 7. 工具类模块 (`utils/`)

**功能**：
- 提供各种通用工具函数，支持项目其他模块

**核心工具**：
- `config_handler.py`：配置文件处理
- `file_handler.py`：文件处理，包括PDF和TXT加载
- `logger_handler.py`：日志处理
- `path_tool.py`：路径处理
- `prompt_loader.py`：提示词加载

## 项目功能总结

1. **基于文档的问答系统**：
   - 能够从PDF和TXT文档中检索相关信息
   - 结合大语言模型生成准确的回答
   - 支持扫地机器人相关知识的查询

2. **多工具集成**：
   - 集成天气查询功能
   - 支持用户位置获取
   - 整合外部系统的使用记录

3. **智能代理**：
   - 基于React模式构建的代理
   - 能够根据用户查询选择合适的工具
   - 支持流式输出回答，提升用户体验

4. **增量学习**：
   - 支持文档的增量添加
   - 使用MD5校验避免重复处理文档
   - 向量数据库持久化存储

5. **可配置性**：
   - 模块化的配置系统
   - 支持不同模型的选择和切换
   - 可调整的向量数据库参数

## 应用场景

1. **扫地机器人知识库**：回答用户关于扫地机器人的问题，包括故障排除、维护保养、选购指南等
2. **智能客服**：提供基于文档的智能客服功能
3. **个人助手**：结合天气查询、位置信息等功能，提供个性化服务
4. **数据分析**：结合外部系统的使用记录，提供数据分析和报告生成

## 技术亮点

1. **模块化设计**：清晰的模块划分，便于维护和扩展
2. **RAG技术应用**：有效结合文档检索和大语言模型，提升回答质量
3. **增量更新机制**：避免重复处理文档，提高系统效率
4. **多工具集成**：扩展代理能力，提供更全面的服务
5. **流式输出**：提升用户体验，减少等待时间

该项目是一个典型的基于RAG技术的智能问答系统，结合了大语言模型和向量数据库，为用户提供基于文档的智能回答服务，特别针对扫地机器人领域的知识查询。
