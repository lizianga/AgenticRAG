# 高效记忆系统设计方案（基于文件系统JSON）

## 一、系统现状分析

通过分析当前项目代码，发现以下现状：

1. **状态管理**：系统使用 `RagState` 类型定义状态，包含 `chat_history` 字段但未实际使用
2. **无记忆机制**：每次调用都是独立的，没有上下文连续性
3. **无用户区分**：系统没有会话管理机制，无法区分不同用户

## 二、记忆系统设计

### 1. 短期记忆存储

**设计方案**：
- **数据结构**：扩展 `RagState` 增加 `session_id` 字段
- **存储方式**：使用内存字典存储对话历史，键为会话ID，值为对话历史列表
- **管理策略**：
  - 设置最大历史长度（如10轮对话）
  - 新对话加入时，超过长度限制的旧对话自动移除
  - 会话超时后自动清理

**实现要点**：
- 在 `generate_node.py` 中修改 `generate_response` 方法，将历史对话作为上下文传递给模型
- 每次生成回答后，更新对话历史

### 2. 长期记忆存储（基于文件系统JSON）

**设计方案**：
- **存储介质**：使用文件系统的JSON文件存储重要对话
- **文件结构**：
  - 目录：`data/memory/`
  - 文件命名：`{session_id}.json`
  - 文件格式：
    ```json
    {
      "session_id": "uuid-string",
      "conversations": [
        {
          "user_query": "用户问题",
          "system_response": "系统回答",
          "timestamp": "2026-04-09T12:00:00",
          "summary": "对话摘要"
        }
      ],
      "last_updated": "2026-04-09T12:00:00"
    }
    ```
- **管理策略**：
  - 定期对对话进行总结，生成摘要存储
  - 使用向量相似度搜索提取相关长期记忆
  - 对长期记忆进行分类和标签化管理

**实现要点**：
- 创建 `memory_service.py` 实现长期记忆的存储和检索
- 集成到 `generate_node.py` 中，在生成回答时考虑长期记忆

### 3. Session 控制模块

**设计方案**：
- **核心类**：创建 `SessionManager` 类管理会话
- **功能**：
  - 会话创建：生成唯一会话ID（使用UUID）
  - 会话获取：根据会话ID获取会话状态
  - 会话删除：清理过期会话
  - 会话超时：设置会话超时时间（如30分钟）
  - 状态持久化：可选的会话状态持久化

**实现要点**：
- 创建 `session_manager.py` 实现会话管理功能
- 在系统入口处集成会话管理，为每个用户请求分配会话ID

## 三、完整实现流程

1. **初始化**：
   - 创建 `SessionManager` 实例
   - 初始化短期记忆存储（内存字典）
   - 初始化长期记忆存储（文件系统目录）

2. **会话处理**：
   - 接收用户请求，检查是否有会话ID
   - 无会话ID则创建新会话
   - 有会话ID则获取现有会话

3. **记忆管理**：
   - 短期记忆：更新对话历史，控制长度
   - 长期记忆：定期总结重要对话，存储到JSON文件

4. **生成回答**：
   - 结合短期记忆（最近对话）
   - 检索相关长期记忆
   - 生成上下文感知的回答

5. **会话清理**：
   - 定期清理超时会话
   - 清理过期短期记忆

## 四、优势与特点

1. **高效性**：短期记忆使用内存存储，访问速度快
2. **持久性**：长期记忆使用文件系统存储，保证重要信息不丢失
3. **个性化**：Session控制模块确保不同用户的记忆隔离
4. **智能性**：结合短期和长期记忆，提供更连贯的对话体验
5. **可扩展性**：模块化设计，易于添加新的记忆管理策略
6. **简单性**：使用JSON文件存储，无需额外数据库依赖

## 五、技术实现建议

1. **依赖库**：
   - `json`：内置库，用于JSON文件操作
   - `uuid`：内置库，用于生成会话ID
   - `os`：内置库，用于文件系统操作
   - `datetime`：内置库，用于时间戳管理
   - `langchain`：用于向量相似度搜索

2. **文件结构**：
   - `agent/memory/`
     - `__init__.py`
     - `short_term_memory.py`
     - `long_term_memory.py`
     - `session_manager.py`

3. **集成点**：
   - 修改 `state.py` 扩展状态定义
   - 修改 `generate_node.py` 集成记忆管理
   - 在系统入口处集成会话管理

此设计方案能够有效提升系统的记忆能力，为用户提供更连贯、个性化的对话体验，同时使用文件系统存储长期记忆，避免了额外的数据库依赖。
